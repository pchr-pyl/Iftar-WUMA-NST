🚀 คู่มือการปรับใช้เว็บไซต์
===========================

📋 การเตรียมตัว:
ตรวจสอบให้แน่ใจว่าคุณมีไดเรกทอรี dist/ แล้ว — ซึ่งประกอบด้วยการสร้างเว็บไซต์ของคุณที่เสร็จสมบูรณ์

💡 ปรับใช้บนแพลตฟอร์มคลาวด์ (แนะนำ):

🌟 Vercel (vercel.com) – ง่ายที่สุด
1. ไปที่ vercel.com และสมัครสมาชิก
2. คลิก "New Project"
3. เลือก "Browse All Templates" → "Static"
4. ลากและวางโฟลเดอร์ dist/ ลงในพื้นที่อัปโหลด
5. ใส่ชื่อโปรเจกต์ คลิก "Deploy"
6. รับ URL HTTPS ฟรีภายในไม่กี่นาที!

☁️ Cloudflare Pages (pages.cloudflare.com) – เร็วที่สุด
1. ไปที่ pages.cloudflare.com และเข้าสู่ระบบ
2. คลิก "Create a project" → "Upload assets"
3. บีบอัดโฟลเดอร์ dist/
4. อัปโหลดไฟล์ zip
5. ตั้งชื่อโปรเจกต์ คลิก "Deploy site"
6. เพลิดเพลินกับการเร่งความเร็ว CDN ทั่วโลก!

🔥 Netlify (netlify.com) – ใช้งานง่าย
1. ไปที่ netlify.com และสมัครสมาชิก
2. ลากและวางโฟลเดอร์ dist/ ลงในพื้นที่อัปโหลดของหน้าแรก
3. รอการอัปโหลดให้เสร็จสิ้น
4. คุณจะได้รับโดเมนแบบสุ่ม (สามารถปรับแต่งได้ในตั้งค่า)

📘 GitHub Pages (pages.github.com) – ฟรีและเชื่อถือได้
1. สร้างรีโพสิทอรีใหม่บน GitHub
2. อัปโหลดเนื้อหาของโฟลเดอร์ dist/ ไปยังรีโพ
3. ไปที่การตั้งค่าของรีโพสิทอรี → Pages
4. เลือก "Deploy from a branch" → main branch
5. เข้าถึงเว็บไซต์ของคุณที่ username.github.io/repository-name

---

🔧 การปรับใช้ขั้นสูง (สำหรับนักพัฒนา):

การปรับใช้อัตโนมัติด้วย Git
1. ส่งโปรเจกต์ทั้งหมด (รวมถึงซอร์สโค้ด) ไปยัง GitHub/GitLab
2. เชื่อมต่อรีโพของคุณกับ Vercel/Netlify
3. กำหนดค่าการสร้าง:
   - คำสั่งสร้าง: npm run build
   - ไดเรกทอรีเผยแพร่: dist
   - เวอร์ชัน Node.js: 16.x หรือสูงกว่า
4. ทุกการส่งจะทริกเกอร์การสร้างและการปรับใช้อัตโนมัติ

การปรับใช้เซิร์ฟเวอร์แบบดั้งเดิม
หากคุณมีเซิร์ฟเวอร์ของคุณเอง:
1. อัปโหลดไฟล์ทั้งหมดในไดเรกทอรี dist/ ไปยังรากเว็บของเซิร์ฟเวอร์ของคุณ
2. ตรวจสอบให้แน่ใจว่าเซิร์ฟเวอร์รองรับการโฮสต์ไฟล์สแตติก
3. เข้าถึงเว็บไซต์ของคุณผ่านโดเมนของคุณ

---

💰 ข้อมูลราคา:
- Vercel: แผนฟรีเพียงพอสำหรับการใช้งานส่วนตัว
- Cloudflare Pages: ฟรีทั้งหมด ไม่มีข้อจำกัด
- Netlify: แผนฟรีรวมถึงแบนด์วิดท์ 100GB/เดือน
- GitHub Pages: ฟรีทั้งหมด แบนด์วิดท์ 100GB/เดือน

---

🎯 คำแนะนำ:
- ผู้เริ่มต้น: Vercel หรือ Netlify (อัปโหลดแบบลากและวาง)
- เน้นความเร็ว: Cloudflare Pages
- โปรเจกต์ระยะยาว: GitHub Pages
- นักพัฒนา: การปรับใช้อัตโนมัติด้วย Git

---

⚡ เคล็ดลับ:
- ทุกแพลตฟอร์มให้ใบรับรอง HTTPS ฟรี
- คุณสามารถผูกโดเมนที่กำหนดเองได้ (บางแพลนต้องชำระเงิน)
- หลังการปรับใช้ ทดสอบเว็บไซต์ของคุณบนอุปกรณ์ต่างๆ

---

🆘 การแก้ไขปัญหา:
1. ตรวจสอบให้แน่ใจว่าคุณอัปโหลดเนื้อหาของ dist/ ไม่ใช่โปรเจกต์ทั้งหมด
2. ตรวจสอบว่า index.html อยู่ในไดเรกทอรีราก
3. ตรวจสอบบันทึกข้อผิดพลาดบนแพลตฟอร์มโฮสติ้ง
4. ตรวจสอบให้แน่ใจว่าไฟล์ทั้งหมดถูกอัปโหลดอย่างถูกต้อง


---

🔗 การตั้งค่า Google Apps Script (สำคัญมาก!)
===========================================

⚠️ ก่อนการ Deploy เว็บไซต์ ต้องตั้งค่า Google Apps Script ก่อน!

📝 ขั้นตอนการตั้งค่า:

1. เปิดไฟล์ docs/apps-script-iftar-registration-updated.js
2. คัดลอกโค้ดทั้งหมด
3. ไปที่ https://script.google.com/
4. สร้าง New Project
5. วางโค้ดและแก้ไข:
   - SLIP_FOLDER_ID (Google Drive Folder ID สำหรับเก็บสลิป)
   - SPREADSHEET_ID (Google Sheet ID สำหรับเก็บข้อมูล)

6. Deploy เป็น Web App:
   - คลิก Deploy → New deployment
   - Type: Web app
   - Execute as: Me
   - Who has access: Anyone ⚠️ สำคัญ!
   - คลิก Deploy
   - คัดลอก Web app URL

7. อัพเดท URL ในโค้ด:
   - เปิดไฟล์ src/config.ts
   - แทนที่ GOOGLE_APPS_SCRIPT_URL ด้วย URL ใหม่
   - บันทึกไฟล์

8. Build ใหม่:
   npm run build

9. Deploy ตามขั้นตอนด้านบน

---

🐛 แก้ไขปัญหา "Network error: Failed to fetch":

ดูรายละเอียดใน TROUBLESHOOTING.md และ GOOGLE_APPS_SCRIPT_SETUP.md

สาเหตุหลัก:
- Google Apps Script ยังไม่ได้ deploy
- URL ไม่ถูกต้อง
- Permission ไม่ถูกต้อง (ต้องเป็น "Anyone")

---

✅ ตรวจสอบว่าทุกอย่างพร้อม:
□ Google Sheet สร้างแล้ว
□ Google Drive Folder สร้างแล้ว
□ Apps Script deploy แล้ว (Who has access: Anyone)
□ URL อัพเดทใน src/config.ts แล้ว
□ Build ใหม่แล้ว (npm run build)
□ ทดสอบการส่งข้อมูลแล้ว
